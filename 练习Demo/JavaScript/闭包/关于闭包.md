# 闭包
## 什么是闭包
- 闭包是内层函数能够访问外层函数的变量，即使外部函数已经执行结束，这些变量也不会被销毁。
## 原理
- 一个函数执行完后，它的局部变量会被标记为“不再需要”，垃圾回收器会把它占用的内存释放掉。
- 如果内部函数被返回、或挂到全局某个地方，并且这个内部函数还引用了外部函数的变量，那么垃圾回收器不会回收那些变量 
## 常用于
- 异步任务（定时器、事件监听、Promise）（回调函数需要记住创建时的状态），防抖 / 节流（需要持久化保存 timer 或 lastTime），柯里化

# 问题拓展
## 闭包一定需要返回函数吗？
- 不一定。把内部函数赋值给全局变量、或挂载到某个对象的属性上，也可以形成闭包。

## 闭包对性能有什么影响？
- 提升代码的复用性；若不释放占用更多内存，手动解除引用设为null

## 模块模式和闭包的关系？
- 模块模式就是利用 IIFE（立即执行函数）加闭包，创建私有变量和公有方法
```javascript
const module = (function() {
  let privateVar = 0;
  function privateMethod() {}
  return {
    publicMethod() { privateVar++; }
  };
})(); // module = { publicMethod: [Function: publicMethod] } 变量私有化了
```

# 拓展
## 作用域链
- 每个函数在定义时，会记录一个 Scope 属性，指向它当前所在的作用域链。当函数执行时，要访问一个变量，会沿着 作用域链 从内向外找，直到全局。

## 内存泄露
- “内存泄漏”指不再需要的内存没有被释放。合理使用不会泄漏；滥用且不注意释放引用导致内存占用升高，这样才叫内存泄露
