# 补充知识

## 显式转换和隐式转换

- **显式转换**：你**主动**在代码中调用方法。
- **隐式转换**：JS 在运算或比较时，**自动**帮你把数据类型转换掉。

### 注意点

- 除了 `+` 有特殊性，其他数学运算符会把两边都转成数字。（数学运算）
- 如果两边类型不同，JS 会尝试转成数字再比较。（比较运算）
- `null == undefined` 为 `true`（特殊规则）。
- `NaN` 和任何值（包括自己）比较都是 `false`。
- 逻辑运算时，会隐式转成布尔值来判断，但**返回值不一定是布尔值**。

## 垃圾回收和内存泄露

- 浏览器的垃圾回收机制
  - 一、垃圾回收核心
    - JS 有自动的垃圾回收机制，定期释放不再使用的变量内存：
    * 全局变量：页面卸载前一直占用内存；
    - 局部变量：函数执行结束后自动回收，但若被闭包引用则不会回收。
  * 二、主要回收方式
    - 标记清除（主流）：
      - 标记 “进入环境”（正在使用）和 “离开环境”（不再使用）的变量，释放带 “离开环境” 标记的变量内存。（比如说函数上下文执行结束后销毁，里面的变量都已离开环境）
    * 引用计数（少用）：
      - 统计变量的引用次数，次数为 0 则回收；易引发循环引用问题（如两个对象相互引用，次数永不为 0），需手动设 null 解除引用。
  - 三、减少垃圾回收的优化
    - 数组：清空时设 length=0 而非赋值 \[]（避免创建新对象）；
    * 对象：复用对象，不再使用时设 null；
    - 函数：循环内的函数表达式尽量抽离到外部复用。
    * 及时清理定时器 / 事件监听

* 哪些情况会导致内存泄漏（本该被回收的数据，因某些原因没有回收一直占用内存）
  - 以下四种情况会造成内存的泄漏：
    - 意外的全局变量： 由于使用未声明的变量会变成全局变量，挂载到全局对象，使这个变量一直存在中无法被回收。
    * 遗忘的定时器或回调函数：定时器 / 事件回调（比如click事件）没清除，里面引用的变量一直被持有，无法回收。
    - 脱离 DOM 的引用： 获取一个 DOM 元素的引用，而后面这个元素被删除，由于一直保留了对这个元素的引用，所以它也无法被回收。
    * 闭包： 不合理的使用闭包，从而导致某些变量一直被留在内存当中。

## 事件冒泡、事件委托、事件捕获

- 事件冒泡与事件捕获：
  - 如果两个父子关系标签绑定同名事件，事件冒泡是从触发事件的最内层元素开始，逐层向上传播到父元素；事件捕获则相反，从最外层父元素开始，逐层向下传播到子元素。
  - 比如给 div 嵌套 span，都绑定 click 事件，冒泡时点击 span 会先触发 span 再触发 div，捕获时则先 div 后 span。事件监听第三个参数设为 true 就是捕获阶段触发，默认 false 是冒泡。
- 事件委托：事件委托就是把事件监听绑定在父元素上，利用事件冒泡，让父元素来处理子元素的事件，减少事件监听数量，优化内存
  - 比如 ul 下有多个 li，不用给每个 li 绑 click，直接绑在 ul 上，通过 event.target 判断是哪个 li 触发的。好处是减少事件监听数量，尤其子元素动态增减时不用重新绑定，更高效。

## 数据类型检测

- 1\. typeof
  - 逻辑：快速判断基础类型，返回字符串类型标识；
  * 优点：快速判断基本类型 Number/Boolean/String/Undefined/Function（唯一能精确判断的引用类型）；
  - 缺点：数组、对象、null 均返回 "object"，无法区分引用类型。
- 2\. instanceof
  - 逻辑：基于原型链，判断对象的原型链上是否存在某个构造函数的原型；
  * 优点：正确判断引用类型（Array/Function/Object）；
  - 缺点：无法判断基本类型（如 2 instanceof Number 返回 false）。

## Vue2和Vue3的响应式

- **Vue 2** 用 `Object.defineProperty` 来**截取**对象的读取和修改（`getter` / `setter`）。
- **Vue 3** 靠 `Proxy` **代理整个对象**（拦截所有操作），能够自动检测属性的新增、删除和数组变化，基本类型用 `ref` 包装成对象，引用类型用 `reactive`（基于 `Proxy`）；
- 两者都是在**访问数据时收集依赖**（如组件首次渲染），在**修改数据时触发依赖更新**。

### 关于收集依赖和依赖触发

- 收集依赖：当一个数据被读取，Vue 会把这个“读取行为”记下来，放进一个“订阅列表”里。（收集的是 **“当数据变化时需要重新执行的函数”，如watch、computed、首次渲染**）→ **谁用了我，我就记下谁。**
- **触发依赖：当这个数据被修改时，Vue 会遍历这个“订阅列表”，通知里面所有记录的地方。**→ **我变了，我就告诉所有记过的人。**
