# 定义

- 作用域是变量、函数和对象的可访问范围，决定了代码哪些地方能访问哪些变量。
- 作用域是**静态的**（词法作用域），在写代码时就决定了，和运行时无关
- 每个函数都会创建一个新的作用域。
- JavaScript 主要有三种作用域：**全局作用域**、**函数作用域**、**块级作用域** 

# 原理
## 编译阶段确定作用域
- JavaScript 虽然说是解释型语言，但执行前会经历编译阶段。引擎会扫描所有变量和函数的声明，将它们登记到对应的作用域中。这个过程叫词法作用域。

## 作用域链
- 当代码要访问一个变量时，引擎会从当前作用域开始找，没找到就去外层作用域找，直到全局作用域。这条链就是作用域链。

## 三大作用域的区别
| 作用域类型 | 范围 | 声明方式 | 特点 |
| --- | --- | --- | --- |
| 全局作用域 | 整个程序 | 不在任何函数/块内声明 | 任何地方都能访问，容易冲突 |
| 函数作用域 | 函数内部 | function 内用 var/let/const | 函数内的变量外部无法访问 |
| 块级作用域 | {} 块内 | let / const | 只在 {} 内有效，var 无视块级 |

# 注意
- 同一个作用域内，函数声明优先于变量声明。

# 问题延伸
### 作用域是静态的，那闭包为什么能“记住”外部变量？
- 闭包内部函数引用了外部变量，导致外部函数执行完毕后，其作用域仍然被内部函数引用，不会被垃圾回收。这就形成了“持久化的作用域”。
### 模块作用域（ES6 module）是什么？
- 每个 ES6 模块文件都有自己独立的作用域，模块内声明的变量默认私有，只有通过 export 导出才能被其他模块访问。
