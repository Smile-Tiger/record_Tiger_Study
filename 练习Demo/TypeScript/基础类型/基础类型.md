## 基础类型

## 定义

**TypeScript 基础类型** 是 JS 数据类型在 TS 中的“静态标签”。

```JavaScript
function greet(name: string) {  // 给 name 贴上 string 标签
    return name.toUpperCase();
}
greet(123); // ❌ 在写代码时（编译前）就直接报红线：类型“number”的参数不能赋给类型“string”的参数。
```

## 高频考点类型

#### **（1）`any` 与 `unknown`（面试必考）**

- “`any` 和 `unknown` 都是 TypeScript 中的顶层类型（top type），可以接收任何值。但它们的核心区别在于**使用方式**：
  - `any` 是 **‘完全放弃类型检查’**，你可以在它身上调用任何方法、访问任何属性，TS 都不会报错。
  - `unknown` 是 **‘安全的 any’**，它同样可以接收任何值，但你不能直接使用它，必须先通过类型收缩（如 `typeof`、`instanceof`、`as` 断言）确认它的具体类型后，才能进行操作。

```JavaScript
let a: any = 10;
a.toUpperCase(); // ✅ 通过（any 允许任何操作，非常危险）

let u: unknown = 10;
u.toUpperCase(); // ❌ 报错：u 是 unknown，不能直接调用方法

// 必须收缩类型
if (typeof u === "string") {
    u.toUpperCase(); // ✅ 通过了类型守卫检查，安全！
}
```

#### **（2）数组与元组（Tuple）**

- **数组**：`类型[]` 或 `Array<类型>`，表示一堆相同类型的集合。
- **元组**：**固定长度**的数组，每一项的类型可以不同（像一个小型对象）。

```JavaScript
let list1: number[] = [1, 2, 3];     // 数字数组
let list2: Array<string> = ['a', 'b']; // 另一种写法

// 元组：第一个是 string，第二个是 number，长度固定为 2
let person: [string, number] = ['张三', 25]; 
// person.push('xxx'); // 虽然 TS 会警告越界，但运行时数组方法可能绕过，日常开发我们只用来表示固定结构，不要 push。
```

#### **（3）联合类型（Union Types）—— 非常实用！**

当一个变量**可能是 A 类型，也可能是 B 类型**时，用 `|`（或）连接。

```JavaScript
let id: string | number; 
id = "abc123"; // ✅
id = 456;      // ✅
id = true;     // ❌ 报错，不支持布尔值
// 使用技巧：既然可能是 string 或 number，调用方法时也要注意做区分

// 还可以给类型起一个类型别名
type ID = string | number;
let userId: ID = 101;
let productId: ID = "prod-001";
```

## 补充知识

- TS 非常聪明，能自动“猜”出类型。**能不写就不写，让 TS 自己推断。**
- 书写函数的时候，形参和返回值也需要加类型，没有返回值写在参数列表括号后面写：void
