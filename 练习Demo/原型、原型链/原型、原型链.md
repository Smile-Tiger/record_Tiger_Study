# 定义

## 原型

- 每个 JavaScript 对象（除了 `null`）都有一个内部链接指向另一个对象，这个被指向的对象就是它的“原型”。原型对象上可以挂载属性和方法，供其他对象共享和继承。

## 原型链

- 当试图访问一个对象的属性时，如果对象本身没有，就会沿着这个内部链接往“原型”对象上找，如果原型也没有，就继续往原型的原型上找……直到找到或者到达尽头（`null`）。

## 关键点

- 每个函数都有一个 `prototype` 属性（用于实例化对象时指定原型）
- 在 JavaScript 中，**几乎所有东西都是对象**
- 每个对象（除了 `null`）都有一个 `__proto__` 属性（内部链接），指向它的原型

## 注意

- 可以通过Object.create(对象)，**创建一个新对象，新对象的原型指向**`create括号内的对象`**，从而实现继承；这个方法还是浅克隆方法；而且通过Object.create(null)这样就创建了一个干净（没有原型继承）的对象**
- 对象的proto属性是拿不到的，只有通过`Object.getPrototypeOf(obj)`拿到，而不是 `obj.proto`
- 所有普通对象的原型链最终都会指向 `Object.prototype`，而 `Object.prototype.__proto__ === null`，即原型链的尽头是 `null`
- 对象.hasOwnProperty('属性名')，通过这个方法来区分对象的属性是原型链继承还是本身的，属性 in 对象，看**自己或原型链上**有没有这个属性。

## 问题延伸

#### **`__proto__` 和 `prototype` 什么时候相等？**

- 对普通对象来说，没有 `prototype` 属性（只有函数有）。正常来说proto是指向上一层的构造函数原型对象，prototype是构造函数的原型对象，函数作为构造函数时，实例的 `__proto__` 指向构造函数的prototype此时就会相等。特殊情况是Function构造函数的prototype和proto是相等的

#### **`class` 语法和原型链的关系？**

```JavaScript
// 是构造函数的语法糖
class Animal { speak() {} }
// 等价于
function Animal() {}
Animal.prototype.speak = function() {};
```
