# 详细解析

## 1. 作用域

- `var`：只有**函数作用域**。在 `if`、`for` 里用 `var`，它会穿透花括号，跑到外面去。
- `let`** / **`const`：**块级作用域**。只能在 `{}` 内部访问，出花括号即失效。

## **2. 变量提升与暂时性死区（TDZ）**

这是最容易出错的点。

- `var`：会提升，且**初始化为 **`undefined`，所以可以在声明前访问（值为 `undefined`）。
- `let`** / **`const`：也会提升，但**不会初始化**。从进入作用域到声明语句之间的区域，被称为**暂时性死区（TDZ）**，在此区域访问变量会直接报错 `ReferenceError`。

## **3. 重复声明与修改**

- `var`：可以重复声明，后一个覆盖前一个（容易出 bug）。
- `let`：不能在同一作用域重复声明。
- `const`：不能重复声明，且**不能重新赋值**（注意：如果 `const` 存的是对象，**可以修改对象内部的属性**，因为指针没变）。

## **4. 挂载到全局对象**

在浏览器环境中：

- `var`：在全局声明的变量，会成为 `window` 的属性。
- `let`** / **`const`：在全局声明的变量，不会挂到 `window` 上，而是在全局环境的另一个“声明环境”中。
