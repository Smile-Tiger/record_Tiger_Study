# This
## 定义
- this 是函数运行时自动生成的、指向“当前执行环境对象”的一个特殊关键字
## 原理
- 函数直接调用，不带任何对象修饰，非严格模式：window；严格模式：undefined
- 函数作为某个对象的属性被调用时，this 指向那个对象（隐式绑定）
- 坑：const fn = obj.sayHi; fn(); 这是独立调用，this 丢失 → 回到默认绑定（即全局）
- 显式绑定（call / apply / bind）强制指定 this 指向谁，优先级高于隐式绑定
- 使用 new 调用函数时，会创建一个新对象，并把 this 绑定到这个新对象上，优先级最高
- 箭头函数的 this 不是由调用方式决定，而是继承自外层，并且不可改变
- 优先级总结：new 绑定 > 显式绑定 > 隐式绑定 > 默认绑定（优先级高的会覆盖this）
## 问题延伸
### this 和词法作用域有什么区别？
- 词法作用域取决于代码在哪里写，静态确定。this 取决于函数在哪里调用，动态确定。

### call、apply、bind 的区别？
- call：立即调用，参数逐个传递，apply：立即调用，参数以数组传递，bind：返回一个新函数，不立即调用，可以后续调用（只绑定一次this）

### 箭头函数能用作构造函数吗？能改变它的 this 吗？

## 拓展
### 词法作用域
- 词法作用域是变量在代码中写的位置，决定了哪些函数可以访问它，跟运行时调用链无关。
