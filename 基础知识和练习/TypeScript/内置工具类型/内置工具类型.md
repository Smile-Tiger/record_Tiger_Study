# 原理：映射类型

```JavaScript
// 公式：遍历 T 的所有键（keyof T），赋值给 P，生成新类型，T传的是类型或者接口，或者是从值中提取的类型（typeof）
type MyNewType<T> = {
  [P in keyof T]: T[P]  // 这就是最基础的“复制”
};
```

## **① `Partial<T>` —— 全部变为可选**

```JavaScript
// 手写实现
type MyPartial<T> = {
  [P in keyof T]?: T[P];   // 加个 ? 就是可选
};

// 效果
interface User { id: number; name: string; }
type UpdateUser = MyPartial<User>; // { id?: number; name?: string; }
```

## **② `Required<T>` —— 全部变为必填**

**场景**：严格模式下，确保某个类型的所有属性都必须传

```JavaScript
// 手写实现
type MyRequired<T> = {
  [P in keyof T]-?: T[P];   // -? 是 TS 专用语法，意思是“去掉可选性”
};
```

## **③ `Readonly<T>` —— 全部变为只读**

**场景**：定义常量配置或防止被意外修改的对象。

```JavaScript
// 手写实现
type MyReadonly<T> = {
  readonly [P in keyof T]: T[P];  // 加 readonly 就是只读
};
```

## **④ `Pick<T, K>` —— 从 T 中挑选几个属性**

**场景**：大对象中只取需要的字段（比如只取用户的 `id` 和 `name`，过滤掉 `password`）

```JavaScript
// 手写实现（K 必须是 T 键的联合类型）
type MyPick<T, K extends keyof T> = {
  [P in K]: T[P];   // 只遍历 K，而不是遍历全部
};

// 效果
type UserPreview = MyPick<User, 'id' | 'name'>; // { id: number; name: string; }
```

## **⑤ `Omit<T, K>` —— 从 T 中剔除几个属性**

**场景**：敏感信息过滤（如剔除 `password` 返回给前端）。

```JavaScript
// Omit 的实现稍微绕一点，先 Pick 出“不属于 K”的属性
// 基础实现（利用 Exclude 排他工具类型）
type MyOmit<T, K extends keyof T> = {
  [P in Exclude<keyof T, K>]: T[P];
};

// 效果
type SafeUser = MyOmit<User, 'password'>; // { id: number; name: string; }
```

## **⑥ `Record<K, T>` —— 构造一个对象类型（键是 K，值是 T）**

**场景**：定义字典、映射表、枚举映射。

```JavaScript
// 手写实现（K 可以是联合类型）
type MyRecord<K extends keyof any, T> = {
  [P in K]: T;   // 遍历 K，但所有值都是 T
};

// 效果：定义路由权限映射
type Page = 'home' | 'about' | 'admin';
type Permission = 'public' | 'private';
const routeMap: MyRecord<Page, Permission> = {
  home: 'public',
  about: 'public',
  admin: 'private'
};
```
