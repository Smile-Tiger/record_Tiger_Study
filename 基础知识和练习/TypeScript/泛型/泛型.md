# 泛型

## 定义

**泛型（Generics）** 是一种在定义函数、接口或类时，**不预先指定具体类型**，而在**使用时再指定类型**的一种特性。

```JavaScript
function getValue<T>(val: T): T { 
  return val; 
}
const result = getValue('hello'); // TS 自动推断 T 为 string，result 也是 string！
// 也可以
const result = getValue<string>('hello')
```

## 原理

TS 的泛型在编译成 JS 时会被**完全擦除**（就像 `interface` 和 `type` 一样），它只是编译阶段的“类型占位符”。

1. **定义阶段（写代码时）**：声明类型变量（如 `<T>`），此时 `T` 是一个“未知的占位符”。
2. **调用阶段（使用时）**：TS 编译器会根据你传入的实参（或显式指定的类型），**自动推断**出 `T` 的具体类型。
3. **检查阶段**：编译器用推断出来的具体类型，替换掉所有 `T` 的位置，检查逻辑是否合理（比如你调用了 `T` 上没有的方法，就会报错）。

## 问题解答

#### **`function log<T>(arg: T): T` 和 `function log(arg: any): any` 有什么区别？**

前者会**保留传入的类型**（如果传入 `string`，返回也是 `string`，TS 会提示字符串方法）；后者会**丢失类型信息**（返回 `any`，后续操作无类型提示）

**泛型和 `any`** 的本质区别是什么？

- `any` 完全关闭类型检查。`any` 会让 TypeScript 在编译阶段完全忽略该值的类型，把它当作纯 JavaScript 处理
- **泛型是‘延迟类型检查’，在定义时先不确定具体类型，但在调用时必须传入具体类型，然后 TS 会根据传入的类型进行完整的类型推断和检查。**

**默认泛型是什么？**

如果泛型不指定的时候，默认是any

## 注意

- TS 有一个非常智能的特性叫 **“类型收窄”** 。当你传入一个具体的字符串 `'hello'` 时，TS 会尽可能地推断出最精确的类型。因为泛型 `T` 没有附加约束（比如 `extends string` 并不会把 `T` 强制扩宽为 `string`），所以 TS 把 `T` 推断为 `'hello'` 这个字面量类型。

```JavaScript
function test<T extends string | number>(arg: T): T {
  return arg;
}
const result = test('hello');
// 鼠标悬停在 result 上，TS 提示：const testResult: "hello"

const result2 = test<string>('hello'); // 强制指定 T 为 string，返回值就是 string
```
